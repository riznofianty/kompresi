ABSTRACT
Nowadays, it is easier to obtain data in digital forms, such as text files, images, audio, or video, resulting in more data that needs to be stored. The data storage capacity as well as the speed of data transmission need to be increased in order to solve the problem we experience today. One of the alternative solution is to compress the file, hence it would be possible to save more storage space and optimize the process of transmitting data. The performance of Even-Rodeh Code Algorithm and Fixed Length Encode Algorithm will be calculated and compared based on the Ratio of Compression, Compression Ratio, Space Saving, Bit Rate, Time of Compression and Decompression. Both of these algorithms are a ‘lossless’ type of compression as no data is lost after the compression process. The file type to be tested for the compression and decompression process is plaintext (* .txt) file. There are two types of string that will be used to test the two algorithms: string that have one type of character (homogeneous) and string that have several types of character (heterogeneous), stored in the plaintext (.txt) file. Based on the compression process on homogoneous string, the performance of Fixed Length Encode algorithm is better than the performance of Even-Rodeh Code Algorithm with an average compression of 12,705% and a faster decompression time with an average of 0.002781 seconds. In addition, the compression process on heterogeneous strings with Fixed Length Encode algorithm also shows a better performance than Even-Rodeh Code algorithm, with an average compression ratio of 50,205% and decompression process time with an average of 0.00688 seconds.

Key Word : Even Rodeh Code Algorithm, Fixed Length Encode Algorithm, Compression, PHP, Text, Web.

Pada penelitian ini, penulis akan membandingkan algoritma mana yang lebih efisien di antara dua algoritma yaitu algoritma Even-Rodeh Code dan algoritma Fixed Length Encode yang digunakan dalam melakukan kompresi data berupa File teks. Aspek-aspek kinerja algoritma yang perlu diperhatikan dalam menentukan algoritma mana yang lebih efisien, antara lain Ratio of Compression (RC), Compression Ratio (CR), Space Saving (SS), Bit Rate, dan Time Process.
Algoritma Even-Rodeh Code dikembangkan oleh Shimon Even dan Michael Rodeh pada tahun 1978, dengan cara mengkodekan setiap karakter dengan menggunakan beberapa rangkaian bit. Pembentukan bit yang mewakili masing-masing karakter dibuat berdasarkan frekuensi kemunculan tiap karakter dimana panjang akan  
ditambahkan 3 (tiga) bit dan menjadi kelompok paling kiri dari kode (Salomon, 2007). Sedangkan Algoritma Fixed Length Encode dikenal sebagai algoritma klasik dalam proses kompresi data (Jamaluddin, 2013). Dengan menggunakan algoritma Fixed Length Encode, setiap karakter direpresentasikan dengan sebuah bit biner yang unik.

1.2	Rumusan Masalah
Berdasarkan latar belakang tersebut, maka rumusan masalah dalam penelitian ini adalah Besarnya ukuran memori sebuah File dan lamanya proses pengiriman File tersebut mengakibatkan dibutuhkannya teknik kompresi data, serta dibutuhkan perbandingan dari algoritma Even-Rodeh Code dan algoritma Fixed Length Encode untuk mengetahui algoritma kompresi mana yang kinerjanya lebih baik. 
1.3	Batasan Masalah
Batasan masalah yang digunakan antara lain :
1.	Algoritma yang digunakan adalah algoritma Even-Rodeh Code dan algoritma Fixed Length Encode (FLE) dalam melakukan kompresi File teks.
2.	Jenis File yang akan dikompresi adalah File teks yang berekstensi *.txt.
3.	Kode teks yang digunakan adalah kode ASCII (American Standard Code for Information Interchange) 256 karakter (8 bit).
4.	Parameter pengukuran untuk menganalisis kinerja kedua algoritma kompresi data ini adalah ukuran File hasil kompresi,   Ratio of Compression (RC), Compression Ratio (CR), Space Savings (SS), Bit Rate, dan Time Process (waktu kompresi dan dekompresi).
5.	Penelitian ini dirancang dengan menggunakan bahasa pemrograman PHP yang berbasis web.
1.4	Tujuan Penelitian
Berdasarkan rumusan masalah di atas, maka tujuan dari penelitian ini adalah untuk mengkompresi File teks dengan menggunakan algoritma Even-Rodeh Code dan algoritma Fixed Length Encode serta menunjukkan hasil perbandingan kinerja mana yang lebih baik antara kedua algoritma tersebut.

1.5	Manfaat Penelitian
Manfaat yang diperoleh dari penelitian ini antara lain :
1.	Menghasilkan sistem yang dapat melakukan kompresi dan dekompresi File teks sebagai hasil implementasi dari algoritma Even-Rodeh Code dan algoritma Fixed Length Encode.
2.	Mengetahui algoritma mana yang memiliki kinerja yang lebih baik antara algoritma Even-Rodeh Code dan algoritma Fixed Length Encode dalam memperkecil ukuran File teks.
3.	Hasil penelitian dapat menjadi rujukan untuk penelitian selanjutnya yang berhubungan kompresi data menggunakan algoritma Even-Rodeh Code dan algoritma Fixed Length Encode.
