Pada Flowchart sistem yang ditunjukkan pada Gambar 3.8, dapat disimpulkan bahwa pengguna memasukkan File txt ke dalam sistem terlebih dahulu. Kemudian, dilakukan proses kompresi dengan algoritma Even-Rodeh Code atau algoritma Fixed Length Encode. Selanjutnya, sistem akan menghitung RC, CR, SS, Bit Rate, dan Time Process. Selanjutnya, sistem akan mengeluarkan hasil kompresi berupa File yang berekstensi *.er atau *.fle sesuai algoritma yang sudah dipilih oleh pengguna pada proses sebelumnya. Kemudian, apabila pengguna menginginkan untuk melakukan dekompresi pada File hasil kompresi tadi, maka pengguna harus memasukkan kembali File hasil kompresi ke dalam sistem. Selanjutnya, sistem melakukan proses dekompresi menggunakan algoritma Even-Rodeh Code atau algoritma Fixed Length Encode. Selanjutnya, sistem akan menghitung lamanya waktu proses dekompresi. Setelah proses dekompresi selesai, sistem akan mengeluarkan hasil dekompresi berupa File *.txt seperti File sebelum dikompresi serta waktu dari proses dekompresi berlangsung. Perancangan User Interface sangat penting dilakukan sebelum membuat suatu sistem atau aplikasi. Dengan adanya perancangan Interface akan mempermudah pengguna dalam menggunakan sistem. Pada sistem kompresi ini, terdapat 5 buah halaman yang akan ditampilkan, antara lain halaman Informasi Sistem, halaman Kompresi, halaman Dekompresi, halaman Bantuan, dan halaman Tentang.
